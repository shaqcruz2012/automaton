Payment Flow (x402)

ⓘ
x402 is an open standard for HTTP-native micropayments using EIP-712 signed USDC transfers on Base. No custodians, no subscriptions — funds go directly to the recipient wallet on-chain. Learn more at x402.org.
01
POST to the endpoint
Make a normal HTTP POST request. The server responds with HTTP 402 Payment Required and a JSON body describing the payment — amount, recipient address, deadline, and accepted token.
02
Sign EIP-712 TransferWithAuthorization
Using the 402 response data, construct an EIP-712 TransferWithAuthorization struct and sign it with your wallet. This authorizes a USDC transfer of exactly the required amount without broadcasting a transaction yet.
03
Retry with X-Payment header
Re-send the identical POST request, this time including the X-Payment header containing a base64-encoded JSON payload with your signature and authorization struct (x402Version, scheme, network, payload).
04
Receive your result
The server verifies the payment via the x402 facilitator at https://x402.org/facilitator, settles the USDC transfer on Base, and returns HTTP 200 with your result. Total added latency: typically under 300ms.
Integration

JavaScript SDK

Copy-paste this helper to call any Datchi endpoint from your app. Requires viem for EIP-712 signing. Install with npm install viem.

JavaScript (ESM) 
import { createWalletClient, http } from 'viem';
import { base } from 'viem/chains';
import { privateKeyToAccount } from 'viem/accounts';

const GATEWAY = 'https://datchi.example.com'; // or http://localhost:7402
const USDC    = '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913';

async function callDatchi(endpoint, body, privateKey) {
  // Step 1: Get payment requirements
  const res402 = await fetch(`${GATEWAY}${endpoint}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
  });

  if (res402.status !== 402) return await res402.json();

  const requirement = await res402.json();
  const accept      = requirement.accepts[0];

  // Step 2: Sign payment
  const account = privateKeyToAccount(privateKey);
  const nonce   = '0x' + crypto.getRandomValues(new Uint8Array(32))
    .reduce((s, b) => s + b.toString(16).padStart(2, '0'), '');

  const now           = Math.floor(Date.now() / 1000);
  const authorization = {
    from:        account.address,
    to:          accept.payToAddress,
    value:       accept.maxAmountRequired,
    validAfter:  String(now - 60),
    validBefore: String(now + accept.requiredDeadlineSeconds),
    nonce,
  };

  const domain = {
    name:              'USD Coin',
    version:           '2',
    chainId:           8453,
    verifyingContract: USDC,
  };

  const types = {
    TransferWithAuthorization: [
      { name: 'from',        type: 'address' },
      { name: 'to',          type: 'address' },
      { name: 'value',       type: 'uint256' },
      { name: 'validAfter',  type: 'uint256' },
      { name: 'validBefore', type: 'uint256' },
      { name: 'nonce',       type: 'bytes32'  },
    ],
  };

  const signature = await account.signTypedData({
    domain,
    types,
    primaryType: 'TransferWithAuthorization',
    message: authorization,
  });

  // Step 3: Retry with payment header
  const payment = btoa(JSON.stringify({
    x402Version: 1,
    scheme:      'exact',
    network:     'eip155:8453',
    payload:     { signature, authorization },
  }));

  const result = await fetch(`${GATEWAY}${endpoint}`, {
    method:  'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Payment':    payment,
    },
    body: JSON.stringify(body),
  });

  return await result.json();
}

// Usage
const summary = await callDatchi('/summarize', {
  content: 'Your text here...'
}, '0xYOUR_PRIVATE_KEY');
