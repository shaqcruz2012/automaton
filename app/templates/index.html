A minimal, privacy-forward invoice/receipt parsing API (MVP: plain text → structured JSON).

Endpoints

  • GET /health — service health
  • POST /parse — parse receipt/invoice text (multipart form: text or file)
  • OpenAPI docs

Quick curl

curl -s -X POST http://localhost:8000/parse \
  -F "text=ACME Supplies\n2026-03-01\nWidget 9.99\nTax 0.80\nTotal 10.79" | python -m json.tool

Data handling & privacy

  • No document storage by default (processed in-memory only).
  • Logs only minimal metadata (request size, content type, short hash prefix) for abuse prevention and debugging.
  • No selling of customer data; no training on customer documents unless you explicitly opt in (future feature).
